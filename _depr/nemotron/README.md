# Nemotron Embedding Fine-Tuning Pipeline

This directory contains the custom implementation of **Stage 0 (Synthetic Data Generation)** and **Stage 1 (Data Preparation)** based strictly on the [NVIDIA Nemotron Embed Recipe](https://github.com/NVIDIA-NeMo/Nemotron/tree/main/src/nemotron/recipes/embed). 

Instead of relying on heavy distributed dependencies or external data designer installations, these scripts seamlessly integrate directly into your existing `aisa` configuration environment (`example-config.toml`, `BaseLLM`, and `Embedder`).

---

## 🏗️ Architecture Overview

The pipeline strictly processes `.md` documents, synthetically extracts high-quality target retrieval datasets, and maps those datasets into the exact Automodel biencoder format required for Nemotron fine-tuning.

### Stage 0: Synthetic Data Generation (`sdg.py`)
This script loads documents and divides them into fixed sentence intervals. It executes three localized LLM generations identical to Nvidia's `retriever-sdg`:
1. **Artifact Extraction:** Maps intrinsic structures (key_concepts, insights, causal relationships).
2. **QA Generation:** Utilizes the extracted semantics to yield complex queries possessing explicit, orthogonal delineations between logical `query_type` and `reasoning_type`. 
3. **LLM-as-a-Judge Evaluation:** Rubrics out each generated pairing on relevance, clarity, and accuracy from 1 to 10.

### Stage 1: Data Preparation (`data_prep.py`)
This script refines the raw generations from Stage 0 into training-ready variables.
1. **Quality Filtering:** Purges any pairs scoring below the minimum threshold (default 7.0).
2. **Hard Negative Mining:** Builds a transient RAM-based vectorized Numpy cache. By utilizing your configured `Embedder`, it computes cross-similarity dot products under an explicit margin (e.g., `< 0.95`) to actively surface adversarial context passages.
3. **Document Unrolling:** Translates combinatorial array mappings (`[doc_1, doc_2]`) strictly into discrete JSON structures (`1:1`) to prevent mapping collision during the fine-tuning phase.
4. **Data Slicing:** Dynamically splits data into 80/10/10 train/val/test allocations and yields natively formatted output including `.jsonl` components for BEIR metrics parsing.

## 🚀 Usage Guide

> [!IMPORTANT]
> The scripts depend on your core `aisa` modules. Be sure to run these from the **root of your repository** with `PYTHONPATH` set to the current directory, utilizing your local `.aisa-venv`.

### Running Stage 0 (SDG)

Run the script against a folder of target context texts (e.g., your Markdown files).

```powershell
$env:PYTHONPATH='.'
.\aisa-venv\Scripts\python.exe .\nemotron\sdg.py --input-dir .\rawdata\parsed-techbriefs\
```

**Command Line Arguments:**
- `--input-dir` *(Required)*: Path to the target directory containing your `.md` files.
- `--config` *(Optional)*: Path to the TOML configuration containing your `[llm]` module specifics. Defaults to `./example-config.toml`.
- `--output` *(Optional)*: File path to dump the raw JSON metadata responses. Defaults to `./output/embed/stage0_sdg/generated_batch.json`.

### Running Stage 1 (Data Prep)

Run the preparation pipeline strictly on the JSON output map that was generated during Stage 0.

```powershell
$env:PYTHONPATH='.'
.\aisa-venv\Scripts\python.exe .\nemotron\data_prep.py --sdg-input .\output\embed\stage0_sdg\generated_batch.json
```

**Command Line Arguments:**
- `--sdg-input` *(Required)*: Direct path pointing to the JSON file generated by Stage 0.
- `--config` *(Optional)*: Path to the TOML configuration dictating `[embedding]` specifics. Defaults to `./example-config.toml`.
- `--output-dir` *(Optional)*: Target directory for the final splits and `eval_beir` schema. Defaults to `./output/embed/stage1_data_prep`.
- `--threshold` *(Optional)*: Minimum overall QA evaluating score to parse into the training loop (Scale 1-10). Defaults to `7.0`.
